On the Subject of Password Mutilator EX

This thing is not as secure as 256-bit security, I think.

Note: This module is in beta phase, and this module will be consistently be improved from time to time.
Last updated: 11/07/2023 02:00 GMT+8

In order to solve the module, a password is required to be entered in the input phase, that is after all the non-ignored modules are solved.

The password required to be entered depends on the number of stages and the password changes every second.

Preface

This module consists of:

  • A display
  • Function key (Fn) that serves as Enter to submit the input
  • Backspace
  • Four directional keys for navigating the input
  • Two timer displays

This module consists of four phases:

  • Pre-stage phase
  • Stage phase
  • Pre-input phase
  • Input phase
  • Post error input phase

Section I: Pre-stage phase

This phase requires user to input activation code, this is to verify the authenticity of the bomb user in order to proceed to the next phase.

This activation code can be found in Password Generator manual. Input this activation code after converting all letters to lower, to proceed to the next phase.

Refer to Appendix II for reference during input.

If any non-ignored module were solved during this phase, alarm will be triggered and strike will be given.

Strikes will also be given every 30 seconds after alarm is triggered.

Section II: Stage phase

This phase will display all the information required for the next phase.

The raw value for the given stage n, Rn, is calculated with the formula below:

Rn = [Rn + (In)] mod 104

Where:
In = The increase factor, which is a constant (102 + 2 ≤ If < 103 - 2)

This increment will happen every second and changes every 300 seconds.

However, given the system is corrupted, the Rn might not be calculated as intended. That includes:-

  1. t ≠ 0 when the stage is generated,
  2. Rn is refreshed once per 0.5 second,
  3. Rn is not shown in default base (10), and
  4. In is not constant and is within ± 2.

The calculated value, which changes every second, can be obtained with the formula as follows:-

Cn = Average In + (2FAST™) mod 9 + CTn+1

The Combined Time when the stage n is generated, can be found by following:

Combined Time, CTn = (BTn + ETn) % 60

Where:
BTn = Bomb time remaining (seconds) when the stage n is started
   (or when pre-input phase starts if it is the last stage)
ETn = Total elapsed time (seconds) when the stage n is started
   (or when pre-input phase starts if it is the last stage)

Finally, the final value is constructed as shown as following:

Fn = [ Cn + t ] mod 94 + 33

Where:
t = Total elapsed time (seconds)

Note: If current stage's final value is equal to the previous stage's, add 1 to Fv. Subtract 1 instead if it goes over 126.

Section III: Pre-input phase

Input activation code Press the Fn button at any time to unlock this phase before time expires. You have 3 minutes until it automatically enters the phase.

Section IV: Input phase

Once the input phase is unlocked, the password will now be stabilized: the modification of password will be halted.

Each password character can be found by information given in the stage phase, where for each stage:

  • Prepend and append &# and ; respectively to all Fn.
  • Convert the result into characters based on their numeric code from HTML Character Codes.
  • Refer to Appendix II for reference during input.
